## 4. 그 이후의 프롬프트

Lovable의 무료 크래딧 사용 초과로 인해 진행이 불가능..

따라서 진행하려 했던 그 이후의 프롬프트를 정리해봄.

---

### ✅ 1. 새 동아리 만들기 버튼 활성화
**의도:** 사용자가 직접 동아리를 개설할 수 있게 함.  
**보완 제안:**
- “공개 / 비공개 / 초대 전용” 선택 옵션 추가  
- 카테고리(전략, 마피아, 파티 등) 설정 가능  
- 생성 시 자동 단체 채팅방 생성  
---

### ✅ 2. 대댓글 기능 활성화
**의도:** 커뮤니티 피드 내 상호작용 강화.  
**보완 제안:**
- 댓글/대댓글 계층 구조 시각적으로 구분 (indent, 색상 등)  
- 알림 기능 추가 (“OO님이 내 댓글에 답글을 달았습니다”)  
- 욕설 필터링 등 기본 moderation 기능 고려  
---

### ✅ 3. 프로필 설정 기능 구현
**의도:** 사용자 개성 및 참여도 시각화.  
**보완 제안:**
- 닉네임, 소개글, 선호 게임, 출석 포인트 표시  
- 프로필 이미지 업로드 및 기본 아바타 제공  
- 공개 범위 설정 (전체 / 동아리만 / 비공개)  
---

### ✅ 4. 동아리/동호회 리더의 단체방 관리 시스템 구현
**의도:** 리더 권한 시스템 도입.  
**보완 제안:**
- 멤버 초대 / 강퇴 / 공지 등록 / 일정 고정 기능  
- 운영진(Role) 부여 기능 (관리자 / 일반 멤버 등)  
- 리더 전용 대시보드 (출석, 활동 통계 등)  
---

### ✅ 5. 단체방 내 멤버 프로필 확인 시스템 구현
**의도:** 참여자 간 관계 강화 및 신뢰 구축.  
**보완 제안:**
- 프로필 클릭 시 미니 팝업 (닉네임, 활동 내역, 출석률 등)  
- ‘함께한 모임 수’, ‘선호 게임’ 표시  
---

### ✅ 6. 고유 코드를 이용한 비공개 동아리 시스템 구현
**의도:** 초대형 커뮤니티 내 비공개 그룹 지원.  
**보완 제안:**
- 초대 코드로만 가입 가능  
- 코드 만료 / 재발급 기능  
- 검색 결과 비노출 설정
